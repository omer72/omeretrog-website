<task type="auto" tdd="true">
  <name>Task 2: Build PortfolioItem component and write unit tests</name>
  <files>src/components/work/PortfolioItem.tsx, tests/portfolio-item.test.tsx</files>
  <action>
1. Create `tests/portfolio-item.test.tsx` FIRST (TDD red phase). Tests should:
   - Import `render, screen` from `@testing-library/react` and `describe, it, expect` from `vitest`.
   - Create a mock `PortfolioItem` object with `permissionConfirmed: true` and all fields populated.
   - Create a second mock with `permissionConfirmed: false`.
   - **PORT-01 test:** "renders ReactCompareSlider when permission confirmed" — render with confirmed mock, assert `screen.getByRole('slider')` exists (react-compare-slider renders an internal `input[type=range]`).
   - **PORT-02 test:** "slider has accessible aria-label" — assert the slider role element has an accessible name containing the client name (use `getByRole('slider', { name: /clientName/ })`).
   - **PORT-03 tests:** "displays client name" — `getByText(mock.clientName)`. "displays description" — `getByText(mock.description)`. "displays result" — `getByText(mock.result)`. "displays role" — `getByText(mock.role)`.
   - **Placeholder test:** "renders placeholder when permission not confirmed" — render with unconfirmed mock, assert `queryByRole('slider')` is null, assert text "coming soon" or similar placeholder is present.

2. Run tests — they MUST fail (red phase): `npx vitest run tests/portfolio-item.test.tsx`.

3. Create `src/components/work/PortfolioItem.tsx` (green phase):
   - Import `ReactCompareSlider`, `ReactCompareSliderImage` from `react-compare-slider`.
   - Import `PortfolioItem` type from `../../content/portfolio`.
   - Props interface: `{ item: PortfolioItem }` (the type, not the component — use `as PortfolioItemType` alias to avoid naming conflict).
   - If `item.permissionConfirmed` is `false`, render a placeholder card: `<article>` with `aria-label="Portfolio: {clientName} (coming soon)"`, an `aspect-video` div with centered "Portfolio item coming soon" text, and just the client name + role below. Add `opacity-60` class. Do NOT render the slider.
   - If `item.permissionConfirmed` is `true`, render: `<article aria-label="Portfolio: {clientName}">` containing:
     a. `aspect-video` wrapper div with `overflow-hidden rounded-lg` for CLS prevention.
     b. `ReactCompareSlider` inside the wrapper with `itemOne` and `itemTwo` using `ReactCompareSliderImage` with src, alt from item data. Add `style={{ width: '100%', height: '100%' }}` on each image and `style={{ objectFit: 'cover' }}`. Set `aria-label="Drag to compare before and after website for {clientName}"`. Set `keyboardIncrement="5%"`.
     c. Below the slider: `h2` with clientName, `p` with role (text-text-muted text-sm), `p` with description (text-text-muted mt-2), `p` with result (font-medium text-accent mt-1).
   - Export as default.

4. Run tests — they MUST pass (green phase): `npx vitest run tests/portfolio-item.test.tsx`.

5. Run full test suite to confirm no regressions: `npx vitest run`.
