<task type="auto">
  <name>Task 1: Bootstrap Astro 5 project with Tailwind v4 and sitemap</name>
  <files>package.json, astro.config.mjs, tsconfig.json, .prettierrc, .prettierignore</files>
  <action>
Create the Astro 5 project using the minimal TypeScript template. Run these commands in the project root (the project IS the root — do not create a nested directory):

1. Initialize Astro with minimal template and strict TypeScript:
   ```
   npm create astro@latest . -- --template minimal --typescript strict
   ```
   If it asks about overwriting, accept — the directory only has .planning and .git.

2. Add Tailwind v4 integration (this installs @tailwindcss/vite and configures astro.config.mjs):
   ```
   npx astro add tailwind --yes
   ```

3. Add sitemap integration:
   ```
   npx astro add sitemap --yes
   ```

4. Install dev tools:
   ```
   npm install -D prettier prettier-plugin-astro @axe-core/cli
   ```

5. Update `astro.config.mjs` to include the `site` key (REQUIRED for sitemap generation):
   ```javascript
   import { defineConfig } from 'astro/config';
   import tailwindcss from '@tailwindcss/vite';
   import sitemap from '@astrojs/sitemap';

   export default defineConfig({
     site: 'https://omeretrog.com',
     integrations: [sitemap()],
     vite: {
       plugins: [tailwindcss()],
     },
   });
   ```
   Note: If `npx astro add tailwind` already configured the vite plugin, just add the `site` key and ensure `sitemap()` is in integrations. Do not duplicate the tailwindcss plugin.

6. Create `.prettierrc`:
   ```json
   {
     "plugins": ["prettier-plugin-astro"],
     "overrides": [
       { "files": "*.astro", "options": { "parser": "astro" } }
     ]
   }
   ```

7. Create `.prettierignore`:
   ```
   dist/
   node_modules/
   .astro/
   ```

8. Verify the project builds: `npm run build`

<task type="auto">
  <name>Task 2: Configure dark premium design token system in global.css</name>
  <files>src/styles/global.css</files>
  <action>
Create `src/styles/global.css` with the complete Tailwind v4 design token system. This file is the single source of truth for all colors, typography, and spacing on the site. No component may use inline color values — everything must reference these tokens.

```css
/* src/styles/global.css */
@import "tailwindcss";

@theme {
  /* -- Surfaces -- near-black, not pure black (reduces halation for users with astigmatism) */
  --color-surface:        #111111;   /* page background */
  --color-surface-raised: #1a1a1a;   /* cards, nav background */
  --color-surface-border: #2a2a2a;   /* borders, dividers */

  /* -- Text -- pre-verified against #111111 background at webaim.org/resources/contrastchecker/ */
  --color-text-primary:   #f0f0f0;   /* on #111111 -> 16.75:1  PASS AA+AAA */
  --color-text-secondary: #a8a8a8;   /* on #111111 ->  5.74:1  PASS AA */
  --color-text-muted:     #767676;   /* on #111111 ->  4.54:1  PASS AA (minimum — do not go darker) */

  /* -- Accent -- brand purple */
  --color-accent:         #7c6af7;   /* on #111111 ->  5.94:1  PASS AA */
  --color-accent-hover:   #9585f8;   /* on #111111 ->  8.15:1  PASS AA+AAA */

  /* -- Typography -- */
  --font-sans: 'Inter', ui-sans-serif, system-ui, sans-serif;

  /* -- Spacing & Radius -- */
  --radius-card: 0.75rem;
  --radius-btn:  0.5rem;
}

@layer base {
  :root {
    color-scheme: dark;
  }
  html {
    background-color: var(--color-surface);
    color: var(--color-text-primary);
    font-family: var(--font-sans);
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
  }
  a {
    color: var(--color-accent);
  }
  a:hover {
    color: var(--color-accent-hover);
  }
}
```

After creating the file, ensure it is imported. If Astro's Tailwind setup created a different CSS file (e.g., `src/styles/global.css` or `src/assets/styles/global.css`), use that path instead — but the content must match the above token system.

Verify by running `npm run build` — the build should succeed with the @theme tokens generating CSS custom properties.
